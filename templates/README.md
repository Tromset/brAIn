# templates/ — Ready-to-copy brAIn files

> 🧠 [Parent hub](../README.md) · [Variables](brain.yaml) · Spec: [../brAIn.md](../brAIn.md)

Copy the [skeleton/](skeleton/README.md) folder into any folder you are brAInifying, then fill in the placeholders. Each template implements one of the [6 rules](../brAIn.md).

## Files

| File | What it is for |
|---|---|
| [skeleton/README.md](skeleton/README.md) | Hub template — rule 1 (folder entry point with link slots) |
| [skeleton/brain.yaml](skeleton/brain.yaml) | Variables template — rule 6 (name, purpose, files, `when_to_read`) |
| [skeleton/file.md](skeleton/file.md) | Content-file template — rules 2–4 (nav header + code in fences) |
| [skeleton/config.json](skeleton/config.json) | Deep-config template — rule 5 (stays JSON, linked from the hub) |

## Subfolders

- [skeleton/](skeleton/README.md) — the four templates above, ready to copy as a block.
